# Editor HUD: политика баннера и inline (доп. к ADR 0085)

**Статус:** продуктово-инженерный норматив (v1)  
**Дата:** 2026-04-26  
**Связь:** [0085](../adr/0085-editor-hud-inline-layer-and-hud-banner.md), [0032](../adr/0032-hud-banner-configuration-and-grammar.md), [ux roadmap editor-forward](../ux/editor-forward-ui-cleanup-roadmap-v1.md), [0021](../adr/0021-pfd-mfd-cockpit-attention-model.md) §9

**Цель:** не иметь **двух громких каналов** с **одной и той же** полноформенной прозой; при этом **не** отменять [0085](../adr/0085-editor-hud-inline-layer-and-hud-banner.md) (баннер — отдельное имя; inline — **Editor HUD** в узком смысле).

---

## Роли (напоминание)

| Канал | Роль | Типичное содержимое |
|--------|------|---------------------|
| **Editor HUD (inline)** | Пространственная правда: *где* в файле, *что* за символ, подчёркивания, Quick Info у указателя | Squiggles, inlay (позже), hit-tested tooltip, gutter |
| **HUD banner (file-level)** | Обзор **по файлу** и file-level сигналы, которые **не** требуют позиции каретки | Счётчики диагностик, краткая сводка, «агент / фоновая задача по этому файлу» (когда появятся) |
| **MFD / Problems** | Список по workspace / фильтрация; навигация | Те же **классы** диагностик, иной **носитель** (см. roadmap §5) |

---

## Правила дублирования (v1)

1. **Полный текст** одной и той же диагностики **не** показывать **одновременно** в **баннере** и **inline tooltip** **и** **развёрнутой** строке MFD, если все три видны; минимум один канал — **сжатый** (счётчик / one-liner / только навигация).
2. **Баннер** — по умолчанию **счётчики** и **короткие** метки (`EditorHudBannerTextComposer`); **не** дублировать **произвольный** LSP `Message` **дословно**, если тот же текст уже в squiggle + hover для активной позиции.
3. **Inline** **первичен** для *локального* разрешения: «что не так *здесь*».
4. **Баннер** **оставляем** как канал (см. 0085 §2.2: опционален, но **не** «выкинуть продукт»): для **at-a-glance** по файлу, доступности, сценариев **без** наведения; выключение через настройки — позже [0032](../adr/0032-hud-banner-configuration-and-grammar.md), см. 0085 открытые вопросы.
5. **Quick Info** (типы, сигнатуры) **не** дублировать в баннере — **только** inline (и при необходимости отдельный popover **по политике** roadmap §6 / [0079](../adr/0079-ide-display-system-ids-overlay-pipeline.md)).

## Следующая инженерная фиксация

- Явные флаги в настройках ([0032](../adr/0032-hud-banner-configuration-and-grammar.md)): `banner_diagnostics_summary` / `inline_*` (когда дозреет конфиг) — **не** в этом v1-доке.
- При изменении разметки баннера — проверка этой политики + сценарии MFD/Forward из roadmap.

---

## Успех

Пользователь и ревьюер могут ответить на вопрос «почему это снова в баннере, если оно в тултипе?» ссылкой на **роль** канала, а не на вкус.
