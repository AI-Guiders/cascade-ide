# ADR 0003: Отдельный UI-режим Debug (не кокпит Power)

**Статус:** Accepted (продуктовое направление); **реализация** — по плану релизов  
**Дата:** 2026-04-02  
**Связь:** [0001-debug-hypotheses-json-storage.md](0001-debug-hypotheses-json-storage.md), [ux/cascade-ide-ui-layout-v1.md](../ux/cascade-ide-ui-layout-v1.md) (текущие Focus / Balanced / Power).

## Контекст

Режим **Power** перегружен телеметрией, trace агента, очередью задач и кокпитом автономного сценария. Сценарий «**в основном отладка**» (брейкпоинты, стек, переменные, гипотезы, воспроизведение бага) не должен требовать включения всего этого слоя. IDE в таком сценарии в первую очередь **наблюдатель и панель управления** для действий агента из Cursor, а не обязательный автономный кокпит внутри приложения.

## Решение

Ввести отдельный **UI-режим Debug** (имя в меню/реестре режимов уточняется при реализации):

- Сфокусированная раскладка под отладку и связанные панели (в т.ч. гипотезы — [ADR 0001](0001-debug-hypotheses-json-storage.md)).
- Скрытие или сильное сокращение Power-специфики (очередь задач кокпита, лишняя телеметрия агента там, где не нужна задаче отладки).
- Сохранение общего принципа [0002-debug-human-agent-parity.md](0002-debug-human-agent-parity.md): состояние отладки одно для человека и MCP.

### Карта видимости панелей (целевое состояние)

Ориентир по зонам главного окна — [ux/cascade-ide-ui-layout-v1.md](../ux/cascade-ide-ui-layout-v1.md). Таблица ниже — не «комментарии», а **два ответа на строку**: что режим Debug **оставляет на экране** и что в этом режиме **не показываем** (относительно Power и частично Balanced). Детали хоткеев и биндингов — в спецификации UI при появлении режима в коде.

| Зона | Показываем в Debug | Не показываем в Debug |
| --- | --- | --- |
| **Task cockpit** (строка под тулбаром) | Заголовок и статус задачи, действия для цикла отладки (сборка, запуск под отладчиком) — как в не-Power режимах | Блок **Autonomous** и остальная разметка кокпита, которая сегодня завязана только на `IsPowerMode` |
| **Solution Explorer** | Дерево решения, навигация по файлам | Очередь **PowerTaskQueueItems** под деревом (в Power — да, в Debug — нет) |
| **Колонка редактора** | 1–2 группы редакторов; панель **стек/переменные** при активной DAP-сессии; вывод сборки — доступен (можно держать компактно) | Третья группа редакторов по умолчанию (как в Power); «островной» Power-хром доков, если он остаётся только для Power |
| **Правая колонка (чат)** | Поле ввода и лента сообщений; **Safety** — компактно (как Focus/Balanced) | **Agent Trace**; **Agent Operations** (роль оператора — у внешнего агента; IDE не дублирует слой) |
| **Полоса телеметрии** | Компактная полоса Build / Test / Debug / Git | Вариант полосы и дубли, которые есть только в Power |
| **Нижняя панель** | Вкладки: **Терминал**, **Build output**, **Git** (как сейчас), **Отладка** и **Тесты** при необходимости; **гипотезы** — когда появится UI по [0001](0001-debug-hypotheses-json-storage.md) | Вкладка **События** по умолчанию не открываем (шум); доступ через «Вид», если нужна лента событий |

### Начальная раскладка (реестр режимов)

При добавлении в `UiModeLayoutRegistry` (или аналога) режим Debug задаётся отдельной `UiModeLayoutSpec`: ожидаемо **дерево включено**, **терминал и вывод сборки включены** (как минимум один из каналов вывода нужен для цикла правка–сборка–запуск), **чат** — по продуктовому решению развёрнут или узкий; **число групп редактора** — 1–2. Точные значения фиксируются в коммите, который вводит режим.

### Открытые вопросы (не блокируют ADR)

- Нужна ли в Debug отдельная вкладка **«Гипотезы»** в нижней панели или достаточно области в правой колонке — решить при реализации [0001](0001-debug-hypotheses-json-storage.md).
- Сохранять ли **Quick Actions** из Balanced в task bar — если мешают, убрать для Debug отдельным флагом режима.

Точная карта контролов и хоткеи — в обновлении `cascade-ide-ui-layout-v1` (или преемника) после появления режима в коде.

## Последствия

- Потребуется расширение `UiModeLayoutRegistry` / аналога и условной видимости, а не только новая вкладка внутри Power.
- Документация режимов обновляется, когда режим появится в коде.

## Отклонённые альтернативы

- **Только Power + новая вкладка** без отдельного режима — отклонено как не дающее нужной изоляции сценария «только отладка».
- **Полное дублирование окна** — не требуется; достаточно режима в том же главном окне.
