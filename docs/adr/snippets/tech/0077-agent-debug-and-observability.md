## 2. Агент, отладка, наблюдаемость

**Отладка** — один слой состояния для **человека и агента**: паритет не «про одинаковые кнопки», а про **общую доступность** фактов (где остановились, какие гипотезы, что видно без прятания всего в нижнюю панель). Режим Debug отделён от «кокпита силы» по смыслу.

**Внешний агент** (ACP, stdio, Cursor CLI) — отдельная ось от **MCP-сервера IDE**; не путать транспорты и доверие.

**Граница хоста** (например Cursor) и **IDE**: восстановление MCP-транспорта, перезапуск, сбои — **наблюдаемы** и для оператора, и для сценария агента; это **не** то же самое, что паритет отладочного стека, но тот же принцип честности контракта.

**Контракт агента** (CLI, снапшоты) держит паритет с тем, что видит MCP по смыслу проверок.
