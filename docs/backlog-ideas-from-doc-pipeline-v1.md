# Бэклог: идеи из пайплайна Markdown-сборки (resume / книга)

**Статус:** не в работе — напоминание на возврат к проекту.  
**Источник:** тот же класс решений, что в репозиториях **`resume`** и **`agents-and-humans-book`** (workspace: `PersonalCursorFolder`, при необходимости путь через junction `Personal Cursor Folder`): `dotnet script`, директивы `INCLUDE` / `INCLUDE_MANIFEST` / `INCLUDE_GLOB`, профили `profiles/<имя>/`, артефакты `build/` и `out/`, цепочка экспорта с деградацией (Pandoc + LaTeX → fallback → Edge для PDF).

## Зачем CascadeIDE

Не про обязательный «экспорт книги из IDE», а про **переносимые паттерны**:

| Идея | Зачем в IDE / агенте |
|------|----------------------|
| **Манифест как упорядоченный список путей** | Очередь файлов для пакетных операций, шаги сценария, сборка составного отчёта из фрагментов без одного монолитного файла. |
| **Профили** (`profiles/<имя>/…`) | Разные пресеты без дублирования всего дерева: задача, аудитория, сокращённый набор артефактов. |
| **Предсказуемое раскрытие** (слева направо по документу) | Идемподентные подстановки; меньше сюрпризов для агента и для пользователя. |
| **Цепочка с деградацией** | Основной бэкенд → запасной → мягкий отказ с понятным сообщением (аналогично экспорту/печати, если появятся). |
| **`dotnet-script` как клей** | Пользовательские сценарии без отдельного `.csproj` — рядом с уже существующей оркестрацией команд. |
| **Разделение `build/` vs `out/`** | Промежуточный канон vs то, что «наружу» — ментальная модель для отчётов и кэшей. |
